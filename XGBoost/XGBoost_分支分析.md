## 三、四种分支的核心区别与相互改进关系

### 1️⃣ gbtree（树基）— 基础模式

```
gbtree → 改进: 加入 Dropout → dart
       → 改进: 加入随机采样 → XGBoost RF 模式
       → 完全不同的思路: 用线性模型替代树 → gblinear
```

| 项目 | 说明 |
|------|------|
| **本质** | XGBoost 的原始/默认分支。使用决策树作为基学习器，通过 **梯度提升**（Gradient Boosting）逐步添加树来修正前一轮的残差。 |
| **核心机制** | 每棵新树拟合前一轮损失的**负梯度**方向，累加所有树的预测得到最终结果。 |
| **损失函数** | 支持自定义，默认用 `softmax`（多分类）或 `reg:logistic`（二分类）。 |
| **特点** | 同时支持一阶导（传统 GBDT）和二阶导（Newton 法）优化，收敛更快、精度更高。 |
| **本任务表现** | **97.08%**，算法族内部最佳。 |

---

### 2️⃣ dart（Dropout 树）— gbtree + 随机丢弃

| 项目 | 说明 |
|------|------|
| **改进来源** | 从 gbtree 演化而来，借鉴了**深度学习中的 Dropout 思想**。 |
| **核心机制** | 每一轮迭代时，以概率 `rate_drop` 随机丢弃已有的一部分树，只保留剩余树进行预测并计算梯度，再训练新树来补偿丢弃树的损失。 |
| **关键参数** | `rate_drop`（丢弃概率，默认 0.1）、`skip_drop`（跳过丢弃的概率，默认 0.5） |
| **与 gbtree 的区别** | gbtree 使用**所有已有树**做预测；dart **随机丢弃部分树**后再训练新树。 |
| **优点** | 打破树之间的强依赖关系，防止过拟合，尤其适合小样本或噪声大的数据。 |
| **缺点** | 丢弃机制使收敛变慢，训练时间大幅增加（本任务耗时 570s，是 gbtree 的 80 倍）。 |
| **本任务表现** | 96.55%，比 gbtree 低 0.53%。数据集样本充足（20000条），过拟合风险小，DART 的正则化反而拖累性能。 |

**一句话**: gbtree + Dropout 正则化 → dart

---

### 3️⃣ XGBoost RF 模式 — gbtree + 随机化采样

| 项目 | 说明 |
|------|------|
| **改进来源** | 将 gbtree 的**梯度提升**思路替换为**随机森林的 Bagging** 思路。 |
| **核心机制** | 仍然使用 gbtree 的树结构，但**取消"拟合残差"的串行逻辑**，改为每棵树独立并行训练，通过 `subsample`（行采样）和 `colsample_bytree`（列采样）引入随机性。 |
| **关键参数** | `num_parallel_tree`（并行树数）、`subsample`（样本采样率）、`colsample_bytree`（特征采样率） |
| **与 gbtree 的区别** | gbtree 是**串行**（后一棵树依赖前一棵树）；RF 模式是**并行**（树之间独立）。 |
| **与 dart 的区别** | dart 是**串行 + 丢弃**；RF 模式是**并行 + 随机采样**。 |
| **优点** | 并行训练速度快（5.95s），比 gbtree 还快。 |
| **缺点** | 缺少梯度提升的"逐步修正"机制，精度略低于 gbtree。 |
| **本任务表现** | 96.55%，和 dart 相同。相比纯 ExtraTrees（97.60%）仍有差距。 |

**一句话**: gbtree 的树结构 + 随机森林的并行采样 → XGBoost RF 模式

---

### 4️⃣ gblinear（线性基）— 完全不同的范式

| 项目 | 说明 |
|------|------|
| **改进来源** | **完全抛弃树结构**，改用线性模型作为基学习器。 |
| **核心机制** | 使用带 L1/L2 正则化的线性回归/分类模型（类似 Logistic Regression + ElasticNet），通过坐标下降优化。 |
| **关键参数** | `reg_lambda`（L2正则化）、`reg_alpha`（L1正则化） |
| **与前三者的本质区别** | gbtree / dart / RF 模式都是**非线性树模型**，gblinear 是**线性模型**。树模型能自动学习特征的交互和非线性关系，线性模型只能学习特征的加权求和。 |
| **优点** | 适合高维稀疏数据（如文本分类的词袋特征），速度快、可解释性强。 |
| **缺点** | 无法处理特征间的非线性关系，在结构化数据上远不如树模型。 |
| **本任务表现** | **46.00%**。字母识别需要复杂的非线性决策边界（26个字母 × 16维特征的空间划分），线性模型到此为止。 |

**一句话**: 用线性模型完全替代树提升 → gblinear（适用场景完全不同）

---

### 关系总结图

```
                  ┌─ 串行提升 ─→  gbtree（🏆 97.08%） ── 串行提升 + Dropout ──→  dart（96.55%）
                  │
XGBoost ──────→   ├─ 并行 Bagging ─→  XGBoost RF 模式（96.55%）
                  │
                  └─ 线性模型 ─→  gblinear（46.00%）← 完全不相关的范式
```

| 对比维度 | gbtree | dart | RF 模式 | gblinear |
|---------|:-----:|:----:|:-------:|:-------:|
| 基学习器 | 树 | 树 | 树 | 线性模型 |
| 训练方式 | 串行 | 串行 | 并行 | 坐标下降 |
| 随机化 | 无 | Dropout | 行/列采样 | L1/L2正则化 |
| 非线性 | ✅ | ✅ | ✅ | ❌ |
| 适合任务 | 通用结构化数据 | 小样本/高噪声 | 需要随机化时 | 高维稀疏数据 |
| 本任务准确率 | 97.08% | 96.55% | 96.55% | 46.00% |
| 训练耗时 | 7.14s | 570.59s | 5.95s | 4.77s |

## 五、结论

| 分支 | 排名 | 推荐场景 |
|------|:---:|---------|
| **gbtree（树基）** | 🥇 最佳 | 通用分类任务，精度与速度平衡 |
| dart（Dropout树） | 🥈 | 小样本/高过拟合风险场景 |
| XGBoost RF 模式 | 🥈 | 需要随机化但不想切换框架时 |
| gblinear（线性基） | ❌ | 不适合此任务，适合高维稀疏数据如文本分类 |
