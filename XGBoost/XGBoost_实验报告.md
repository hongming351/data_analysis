## 三、四种分支算法对比

XGBoost 提供了 4 种内置 `booster`，本质上是 3 种不同的算法范式。

### 3.1 总体对比

| 排名 | 分支算法 | `booster` 参数 | 测试准确率 | F1 分数 | 训练耗时 |
|:---:|---------|:------------:|:---------:|:-------:|:-------:|
| 🥇 | **gbtree（树基）** | `'gbtree'` | **97.08%** | **0.9708** | **7.14s** |
| 🥈 | **dart（Dropout树）** | `'dart'` | 96.55% | 0.9655 | 570.59s |
| 🥈 | **XGBoost RF 模式** | `'gbtree'` + RF参数 | 96.55% | 0.9655 | 5.95s |
| 4 | **gblinear（线性基）** | `'gblinear'` | 46.00% | 0.4156 | 4.77s |

> 统一参数: n_estimators=300, learning_rate=0.2, 相同随机种子

### 3.2 各分支详情

#### 🥇 gbtree（树基）— 默认分支

```
booster = 'gbtree'
参数: max_depth=6, subsample=0.8, colsample_bytree=0.8
```

| 指标 | 值 |
|------|:---:|
| 测试准确率 | **97.08%** |
| F1 分数 | 0.9708 |
| 训练耗时 | **7.14s** |
| 训练方式 | 串行梯度提升 |

**优点**: 精度与效率均衡最佳，适合通用结构化数据  
**缺点**: 高噪声场景可能过拟合

> **结论**: 🏆 **XGBoost 家族最佳分支**

---

#### 🥈 dart（Dropout 树）— 带正则化的 gbtree

```
booster = 'dart'
参数: max_depth=6, rate_drop=0.1, skip_drop=0.5
```

| 指标 | 值 |
|------|:---:|
| 测试准确率 | 96.55%（比 gbtree 低 0.53%） |
| 训练耗时 | **570.59s**（gbtree 的 80 倍） |
| 与 gbtree 的关系 | gbtree + **Dropout 正则化** |

**机制**: 每轮随机丢弃已有树的一部分（`rate_drop`），打破树间依赖，降低过拟合  
**优点**: 小样本、高噪声数据上优于 gbtree  
**缺点**: 丢弃机制导致收敛极慢，耗时是 gbtree 的 80 倍  
**本任务**: 数据集过拟合风险小，Dropout 反而拖累性能

> **适用场景**: 小样本、高噪声数据

---

#### 🥈 XGBoost RF 模式 — 并行化的 gbtree

```
booster = 'gbtree'
参数: num_parallel_tree=300, subsample=0.8, colsample_bytree=0.8
（不使用梯度提升，采用 Bagging 策略）
```

| 指标 | 值 |
|------|:---:|
| 测试准确率 | 96.55%（与 dart 相同） |
| 训练耗时 | **5.95s**（四种分支中最快） |
| 与 gbtree 的关系 | gbtree + **Bagging 随机采样** |

**机制**: 将 gbtree 的串行提升改为**并行 Bagging**，每棵树独立训练，通过行/列采样引入随机性  
**本质区别**: gbtree 是**串行 + 梯度提升**，RF 模式是**并行 + Bagging**  
**优点**: 并行训练速度快，适合大规模数据  
**缺点**: 缺少梯度提升的逐步修正机制，精度略低于 gbtree

> **适用场景**: 需要随机化但不想使用 sklearn RandomForest 时

---

#### 4️⃣ gblinear（线性基）— 不同范式

```
booster = 'gblinear'
参数: reg_lambda=1.0, reg_alpha=0.0
```

| 指标 | 值 |
|------|:---:|
| 测试准确率 | **46.00%**（显著低于树分支） |
| F1 分数 | 0.4156 |
| 训练耗时 | 4.77s |
| 与前三个的本质区别 | **线性模型 vs 树模型** |

**机制**: 使用带 L1/L2 正则化的线性模型（类似 Logistic Regression + ElasticNet），通过坐标下降优化  
**优点**: 适合高维稀疏数据（如文本分类的词袋特征）  
**缺点**: 无法处理特征间的非线性交互关系  
**本任务**: 字母识别需要复杂的非线性决策边界，线性模型容量严重不足

> **适用场景**: 高维稀疏数据（文本分类等），不适用于结构化表格数据

## 四、分支间的改进关系分析

### 4.1 演化关系图

```
                          ┌─ 改进: 串行提升 + Dropout ──→  dart（96.55%）
                          │
XGBoost ──→  gbtree ──────┼─ 改进: 改为并行 Bagging  ──→  XGBoost RF 模式（96.55%）
  核心算法    （97.08%）   │
     ↑                    └─ 不同范式: 线性模型替代树 ─→  gblinear（46.00%）
     │
  支持所有分支
```

### 4.2 改进维度对比

| 对比维度 | gbtree | dart | RF 模式 | gblinear |
|---------|:-----:|:----:|:-------:|:-------:|
| **基学习器** | ✅ 决策树 | ✅ 决策树 | ✅ 决策树 | ❌ 线性模型 |
| **训练方式** | 串行提升 | 串行 + 丢弃 | **并行 Bagging** | 坐标下降 |
| **随机化策略** | 无 | Dropout 丢弃 | 行/列采样 | L1/L2 正则化 |
| **非线性能力** | ✅ 强 | ✅ 强 | ✅ 强 | ❌ 弱 |
| **训练速度** | ⚡ 快 | 🐢 极慢(570s) | ⚡ 最快(5.95s) | ⚡ 快 |
| **测试准确率** | **97.08%** 🏆 | 96.55% | 96.55% | 46.00% |
| **适合场景** | 通用结构化数据 | 小样本/高噪声 | 需随机化场景 | 高维稀疏数据 |

### 4.3 改进关系总结

| 改进路径 | 从 → 到 | 改进方式 | 效果 |
|---------|:-------:|---------|:----:|
| **路径 1** | gbtree → **dart** | 加入 Dropout 丢弃机制 | 防止过拟合，但训练耗时激增 |
| **路径 2** | gbtree → **XGBoost RF 模式** | 串行→并行，引入 Bagging | 速度更快，精度略降 |
| **路径 3** | gbtree → **gblinear** | 树模型→线性模型（完全不同） | 精度大幅下降，换场景适用 |

- **路径 1 和 2** 是在 gbtree 基础上的增量改进，保持了树模型的非线性能力
- **路径 3** 是完全不同的算法范式，适用于完全不同的数据特征

## 五、模型总结

### 5.1 XGBoost 整体表现

| 评价维度 | 表现 |
|---------|:----:|
| 最佳准确率 | **97.08%**（gbtree 分支） |
| 在 5 大算法族中排名 | **第 4 名**（SVM > 树集成 > MLP > **XGBoost** > KNN） |
| 训练效率 | 较低（GridSearch 需 160s，单次 7s） |
| 调参复杂度 | **高**（54 种参数组合搜索） |

### 5.2 各分支推荐场景

| 分支 | 推荐场景 | 理由 |
|------|---------|------|
| **gbtree** 🥇 | **通用分类任务首选** | 精度最高（97.08%），速度适中（7s） |
| **dart** | 小样本、高噪声数据 | Dropout 机制有效防止过拟合 |
| **RF 模式** | 需要并行加速的场景 | 训练最快（5.95s），精度尚可 |
| **gblinear** | 高维稀疏数据（文本分类） | 线性模型+L1/L2正则化，不适合表格数据 |

### 5.3 关键发现

1. **gbtree 是最优分支**，精度（97.08%）与速度（7.14s）的综合性价比最高
2. **DART 不适合此任务**：数据集过拟合风险小，DART 的 Dropout 机制无效，且训练耗时是 gbtree 的 80 倍
3. **XGBoost RF 模式不如纯 ExtraTrees**：同样并行 + 随机化的思路，ExtraTrees（97.60%）优于 XGBoost RF（96.55%）
4. **gblinear 不适合结构化数据**：46% 的准确率证明线性模型无法胜任非线性分类任务
5. **边缘特征最关键**：`x_ege`（水平边缘计数）以 12.1% 的重要性位居第一
